Privacy Policy

This Privacy Policy explains how Ephemeral Notes ("the Application") collects, uses, and stores information. By using the Application you agree to the practices described here.

Data Collected

  • Note content: When you create a note, its content is stored temporarily in Redis with an expiration (TTL). The Application may optionally store a hashed password for password-protected notes (passwords are not stored in plaintext).
  • Metadata: The Application maintains lightweight creation metadata (creation timestamp, token identifier, TTL, and flags like markdown/burn-after-read) for display in the Recent Notes list and counts. This metadata does not include note content.
  • Logs: Standard server logs (access logs, error logs) may record request metadata such as timestamps and IP addresses depending on your deployment. The example development server does not persist advanced analytics by default.

How Data Is Used

Note content is stored solely to enable the application to deliver the note to an authorized viewer before it expires. Metadata is used to display recent notes and total counts in the admin UI. Passwords are used only to gate access and are not used for any other purpose.

Retention and Deletion

Notes are retained only until their configured TTL expires or until they are deleted by a burn-after-read event. Creation metadata is retained in Redis for a limited recent history (configurable in the application). If you self-host the application, retention is controlled by your Redis configuration and any additional backups you configure.

Security

The Application includes several safeguards to reduce risk, but no system is perfect. Recommended deployment practices:

  • Run the application behind HTTPS (TLS) to protect data in transit.
  • Use a properly configured Redis instance with network access controls and, if available, authentication and encryption at rest.
  • Limit access to the administration or debugging interfaces in production environments.
  • Consider client-side encryption if notes contain highly sensitive content; the Application does not provide end-to-end encryption by default.

Third-Party Services

Ephemeral Notes uses Redis for temporary storage and may use other libraries (Flask, Markdown, Bleach) for functionality. These dependencies are local to your deployment when self-hosted and do not imply external data sharing by the application itself.
